AWS S3 Access
Control Demonstration

Three asset tiers — each governed by a distinct S3 bucket policy — showcasing public access, IAM-restricted private access, and IP-whitelisted conditional access.

● Public Access
🌐
AWS S3 · Public Bucket Policy
Global Cloud Dashboard

A publicly accessible asset delivered via S3 static hosting. The bucket policy grants s3:GetObject to all principals (*), making it accessible to any user worldwide with zero authentication required.

🔒
Expected Behavior Image loads flawlessly for every visitor — no credentials needed. Served directly from the S3 endpoint.
📸 Open Public Asset
● Private · 403
🔒
AWS S3 · No Public Policy
Internal Financial Architecture

A private, restricted asset with no bucket policy granting public access. Direct HTTP requests receive a 403 Access Denied response from S3 — access requires valid AWS IAM credentials or a pre-signed URL with a time-limited signature.

⚠
Expected Behavior S3 returns HTTP 403 Forbidden. The new tab will show AWS's Access Denied XML error page.
🔒 Attempt Private Access
● IP-Restricted
👥
AWS S3 · IP Condition Policy
Secure Core Network Database

Access is governed by an S3 bucket policy using an aws:SourceIp condition. The asset is accessible only from a whitelisted public IP address (e.g., your mobile hotspot). All other source IPs receive a 403 Denied response instantly.

🌐
Expected Behavior Opens only from the whitelisted IP. Any other network returns HTTP 403 Forbidden from S3.

About This Lab — AWS S3 Static Website Hosting

This entire website is hosted on Amazon S3 as a static website. The three asset cards above demonstrate S3 bucket policy access control tiers: (1) a public policy allowing anonymous reads, (2) no policy resulting in a 403 denial, and (3) a conditional policy using aws:SourceIp to whitelist a specific IP range. The error.html file is configured as the custom error document in the S3 static website settings.

Platform Capabilities

Built on AWS-Grade
Security Primitives

Every layer of NexusCloud is engineered with defense-in-depth — from storage-level policies to edge-network controls.

🔒
Bucket Policy ACLs

Fine-grained resource-level policies on every S3 object — control who can read, write, or list assets with surgical precision using IAM JSON policies.

🌐
IP Allowlisting

Restrict asset access to specific CIDR ranges using aws:SourceIp conditions — perfect for corporate networks and mobile hotspot whitelisting.

🔗
Pre-signed URLs

Grant time-limited, cryptographically signed access to private objects without modifying bucket policies — ideal for temporary file sharing workflows.

📊
S3 Access Logs

Every request — including 403 denials — is captured in S3 server access logs for compliance, forensics, and anomaly detection pipelines.

☁
CloudFront CDN

Overlay a CloudFront distribution for global edge caching, custom domains, HTTPS enforcement, and WAF integration on any S3 origin.

🛡
MFA Delete

Enable MFA-protected deletion on versioned buckets to prevent accidental or malicious data loss — a critical last line of defense for production data.
